# PR Reviewer

PR の差分を対象に、以下の観点でレビューする。

## ファイル・命名規約

- ファイル名がケバブケース（kebab-case）になっているか。PascalCase は NG
- 新規コンポーネントが正しいディレクトリに配置されているか
  - `src/components/ui/` — shadcn 生成のみ
  - `src/components/common/` — 汎用コンポーネント
- import alias が `@/*` を使っているか（`#/*` や相対パスの深いネストは NG）

## コンポーネント規約

- `@unpic/react` を直接 import していないか（`common/media/image.tsx` 経由で使用する）
- `src/components/ui/` のファイルを手動編集していないか
- 新規の汎用コンポーネントに stories が含まれているか
- stories の title がディレクトリ構造と一致しているか（`Common/{Category}/{Name}`）

## Tailwind CSS

- tailwind-reviewer スキルの観点に準拠しているか
- モバイルファーストになっているか
- カスタムプロパティ（`styles.css` で定義）を適切に使用しているか
- `@apply` の乱用がないか

## 型安全性

- `any` の使用がないか。ある場合は妥当な理由があるか
- 型アサーション（`as`）が最小限か
- props の型が export されているか

## テスト

- 影響範囲の大きい変更（共通コンポーネント、ユーティリティ等）にテストが追加または更新されているか
- 既存テストが壊れていないか

## セキュリティ

- 環境変数やシークレットがハードコードされていないか
- ユーザー入力の sanitize が必要な箇所で対応されているか

## その他

- 不要なファイル（デバッグ用コード、console.log 等）が残っていないか
- コミットメッセージが変更内容を適切に反映しているか
- breaking change がある場合、README や CLAUDE.md が更新されているか

## レビュー出力

問題を発見した場合、ファイルパスと行番号を示し、理由と修正案を提示する。重要度を以下で分類する:

- **must** — マージ前に修正が必要
- **should** — 強く推奨。理由があればスキップ可
- **nit** — 軽微な指摘。対応は任意
